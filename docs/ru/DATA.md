# Data sources & schema

> RU: перевод английского оригинала (DATA.md). При расхождении английская версия — источник правды.

> Что представляет собой каждый источник, откуда берутся его числа и в какую унифицированную
> запись всё нормализуется. **Реальные и синтетические данные помечаются для каждого источника** и никогда не смешиваются.

## Sources

В задании названы четыре входных источника. Каждый из них мы наполняем максимально реальными данными, которые можем честно получить:

| Source file | Что представляет | Как мы это получаем | Реальность |
|-------------|--------------------|------------------|---------|
| `google_ads_keywords` | Ключевые слова Site.pro, уже используемые в Ads | Keyword Planner, с посевом по URL `site.pro` | **Реальные** идеи + метрики (ассоциации Google для домена — прокси для живого аккаунта, помечено как таковое) |
| `search_console_queries` | Реальные запросы Search Console для Site.pro | Нет доступа к их Search Console → **синтетика** (реалистичная структура); адаптер API запланирован | **Синтетика** |
| `ahrefs_organic_keywords` | Собственные органические ключевые слова Site.pro | Keyword Planner по `site.pro` (нет подписки Ahrefs) | **Реальные** метрики через Keyword Planner |
| `ahrefs_paid_keywords` | Платные ключевые слова конкурентов | Keyword Planner, с посевом по URL доменов конкурентов | **Реальные** идеи конкурентов + метрики |

**Что мы намеренно не подделываем:** приватные, специфичные для аккаунта данные Site.pro — их
живой список ключевых слов Ads, их фактические запросы Search Console и их подписку Ahrefs. У нас
нет этих аккаунтов, поэтому они представлены как чётко помеченные синтетические файлы с
реалистичной структурой. Шов `ApiAdapter` (см. [`API.md`](API.md)) заменяет синтетику на
реальный поток, как только Site.pro предоставит доступ.

## Provenance of real metrics

Реальные **объём** поиска, **CPC** и **конкуренция** берутся из **Google Ads Keyword Planner**
(через наш доступ к Google Ads API), с посевом по URL домена, чтобы собрать ключевые слова, которые Google
ассоциирует с ним. Примечания по интерпретации:

- `avg_monthly_searches` — среднее за 12 месяцев, разбитое по диапазонам/округлённое Google. Трактовать как относительное,
  а не точное.
- `competition` — конкуренция рекламодателей (LOW/MEDIUM/HIGH), **а не** сложность SEO.
- CPC low/high — диапазон коммерческой ценности; высокий CPC при низком объёме всё ещё может быть достоин ставки.

Метрики получаются на этапе сборки и записываются во входные файлы; развёрнутое приложение только
импортирует файлы и не хранит **никаких учётных данных**.

## Input columns per source

Адаптеры принимают распространённые форматы экспорта; неизвестные столбцы игнорируются, отсутствующие обязательные
приводят к сбою пакета с понятным сообщением.

- **google_ads_keywords** — `keyword`, `avg_monthly_searches`, `competition`, `cpc`,
  `match_type`, `campaign`, `status` (used/paused)
- **search_console_queries** — `query`, `clicks`, `impressions`, `ctr`, `position`
- **ahrefs_organic_keywords** — `keyword`, `volume`, `kd`, `cpc`, `position`, `url`, `traffic`
- **ahrefs_paid_keywords** — `keyword`, `volume`, `cpc`, `competitor_domain`, `url`

Там, где в источнике отсутствует столбец языка/рынка, язык определяется путём распознавания.

## The generated dataset (`sample-data/`)

Четыре входных источника генерируются и коммитятся в `sample-data/` (с
JSON-вариантом **каждого** источника в `sample-data/json/`) — готовы к загрузке в админ-панели:

| File | Rows | Real / sample |
|------|-----:|---------------|
| `google_ads_keywords.csv` | 52 | реальные объём/CPC (уже используемый набор) |
| `search_console_queries.csv` | 79 | синтетические метрики, реальные термины |
| `ahrefs_organic_keywords.csv` | 136 | реальные объём/CPC; KD/поз./трафик — синтетика |
| `ahrefs_paid_keywords.csv` | 112 | реальные объём/CPC конкурентов |

Реальные метрики берутся из Google Ads Keyword Planner по шести языкам (en, de, es, fr, pt,
it) и пяти доменам конкурентов. Набор намеренно содержит брендовые термины, дубликаты, мусор,
строки с низким объёмом и запрещённые примеры, чтобы каждое правило очистки было задействовано. Подробности:
`sample-data/README.md`.

## Unified `keyword` schema

Каждый источник отображается на эту запись; представления админ-панели и экспорт читают из неё.

| Field | Значение |
|-------|---------|
| `batch_id` | пакет импорта, из которого она пришла (FK → `import_batch`, каскад) |
| `source` | `google_ads` \| `search_console` \| `ahrefs_organic` \| `ahrefs_paid` |
| `raw_term` | термин точно в том виде, в каком он был импортирован |
| `normalized_term` | в нижнем регистре, обрезанный, со схлопнутыми пробелами, с сортировкой токенов (ключ дедупликации) |
| `language`, `geo` | язык / рынок (из источника или определённый для Search Console) |
| `avg_monthly_searches` | месячный объём поиска (реальный, где доступен; null для Search Console) |
| `cpc` | цена за клик (одно значение на ключевое слово; null там, где в источнике его нет) |
| `competition` | конкуренция рекламодателей — `LOW` \| `MEDIUM` \| `HIGH` (только Google Ads) |
| `competitor_domain` | задаётся для ключевых слов конкурентов (платных) |
| `source_url` | целевой/посадочный URL из источника, если присутствует |
| `clicks`, `impressions`, `position` | дополнительные метрики, переносимые из Search Console / Ahrefs organic |
| `raw_data` | полная исходная строка источника, в формате JSON, для аудита |
| `is_junk`, `is_duplicate`, `is_brand`, `below_volume` | флаги очистки (задаются на этапе 4) |
| `is_already_used`, `is_forbidden` | флаги подготовки (задаются на этапе 5) |
| `stage` | `imported` \| `cleaned` \| `prepared` \| `ad_ready` |
| `drop_reason` | почему запись была исключена (управляет воронкой) |
| `dedup_group_id` | каноническая группа для объединённых дубликатов |

Таблица `import_batch` записывает по одной строке на каждую загрузку: `source`, `filename`, `format` (csv|json),
`rows_total` / `rows_imported` / `rows_skipped`, `status` (imported|failed) и `message`.

## Language & target URL

- Ключевые слова группируются по `language`/`market` как для текста объявлений, так и для структуры кампаний. Три
  источника несут явный столбец языка; **у Search Console его нет, поэтому язык
  определяется** небольшим детектором маркерных слов/диакритики (по умолчанию английский, когда ничего
  отличительного нет — см. `services/LanguageDetector.php`).
- Каждый язык сопоставляется с корректным локализованным посадочным URL Site.pro. Проверено вживую (2026-07-01):

  | Language | Landing URL |
  |----------|-------------|
  | en | `https://site.pro/` |
  | de | `https://site.pro/de/` |
  | es | `https://site.pro/es/` |
  | fr | `https://site.pro/fr/` |
  | it | `https://site.pro/it/` |
  | pt | `https://site.pro/pt-br/` (страницы `/pt` не существует; сайт отдаёт португальский на `/pt-br/`) |

  Эта карта находится в `backend/config/params.php` и может быть переопределена в админ-панели, как только
  будут предоставлены канонические URL (используется начиная с этапа 6).
